# Documentação de referência da API

Para a referência da API, utiliza-se o [Scalar](https://www.scalar.com/), juntamente com o [Laravel Scribe](https://scribe.knuckles.wtf/laravel/).

Aqui está uma documentação de como configurar o Scalar com o Laravel Scribe no seu projeto: [https://guides.scalar.com/scalar/scalar-api-references/scalar-api-reference-for-laravel-scribe#set-up-laravel-scribe](https://guides.scalar.com/scalar/scalar-api-references/scalar-api-reference-for-laravel-scribe#set-up-laravel-scribe)

Após configurado, na rota `/docs` da sua API, você terá acesso a uma documentação completa e repleta de funcionalidades.

Por padrão o Laravel Scribe documenta automáticamente os paramentos que devem ser enviados para o endpoint, através da [leitura do validador](https://scribe.knuckles.wtf/laravel/documenting/query-body-parameters#examples)
definido no controller, documente a sua API com mais detalhes utilizando-se de annotations, comentários e configurações: [https://scribe.knuckles.wtf/laravel/documenting](https://scribe.knuckles.wtf/laravel/documenting).

> Provavelmente será exigido no momento do Review que a api esteja bem documentada.
