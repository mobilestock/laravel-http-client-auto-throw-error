# Documentação criar um componente.

#### Por que criar um componente:

1. Objetivo claro ( Não criar componente sem motivo ).
2. Reutilização de código.
3. Separação de responsabilidades.
4. Centralização de regras.
5. Facilitar o uso.
6. Manutenção isolada.

#### Regras para criar um componente:

1. A guia de estilo do componente deve ser compatível com o utilizador não o obrigando a ter que imbuir um estilo prévio para que seja renderizado.
2. Deve ser extensível ao máximo.
3. Não deve prejudicar a exibição de componentes filhos.
4. Deve ser de fácil uso.
5. Deve ter documentação.
6. Deve possuir testes.
7. Presar pela responsabilidade unica, salvo exceções de utilizações comuns ( exemplo do formInput )
